Enhancement Three

Databases

Databases

Artifact Overview

The selected artifact for this enhancement is my “Recipe Finder” application, originally created in a previous course and refined throughout CS-499. This React-based web application allows users to search for recipes using the MealDB API. For the database-focused enhancement, I introduced a backend powered by Node.js, Express, and MongoDB to support a new user review system. This version was developed during Module Five and is hosted on both Netlify (frontend) and Render (backend), with a functional MongoDB Atlas cloud database.

Justification for Inclusion in ePortfolio

I selected this artifact for my database enhancement because it offered a practical and user-facing opportunity to implement full CRUD (Create, Read, Update, Delete) operations using a real NoSQL database. The implementation showcases essential database skills, including schema design, backend routing, RESTful API development, and secure interaction with a frontend React app. Specifically, I replaced the previous static JSON data model with dynamic data storage using MongoDB, enabling real-time review persistence across sessions and devices.

How the Artifact was Improved

Originally, recipe reviews were stored in a local JSON file. In this enhancement:

  • I created a backend with Express and connected it to MongoDB Atlas using Mongoose.
  • Defined a Review model schema to include recipe ID, user email, user name, rating, comment, and timestamp.
  • Built API endpoints to fetch, post, update, and delete reviews (GET, POST, PUT, DELETE).
  • Updated the frontend to display reviews per recipe from MongoDB using dynamic fetching.
  • Enabled logged-in users to submit new reviews, as well as edit or delete their own.
  • Replaced previous hardcoded values with real user authentication context using Google OAuth.
  • Included client-side sorting (by rating or date) for improved data accessibility.

    • Reflection on the Enhancement Process

    Transitioning from static JSON files to MongoDB required thoughtful backend design and a deeper understanding of RESTful API structures. One challenge was ensuring proper authentication and ownership verification for user-specific review actions. Another was managing asynchronous data fetching and state updates within React.

    Through this process, I gained valuable experience in:

  • Structuring NoSQL databases for scalable, user-generated content.
  • Writing secure and maintainable backend routes.
  • Connecting frontend components to cloud-hosted databases.
  • Deploying and managing full-stack applications using Netlify and Render.
