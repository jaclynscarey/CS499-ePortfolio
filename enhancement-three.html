Enhancement Three

Databases

For this enhancement, I transformed the Recipe Finder App by replacing the static JSON data storage with a MongoDB database. This enhancement falls under the Databases category by demonstrating the implementation of a NoSQL database solution for scalable data storage and retrieval.

The enhancement required setting up a MongoDB Atlas cluster, creating appropriate data models, and implementing CRUD operations through a RESTful API. I designed the database schema to efficiently store recipe data, user information, and reviews, while ensuring proper indexing for optimal query performance.

These changes not only improve the application's scalability and data management capabilities but also showcase my ability to implement modern database solutions in a full-stack application. The enhancement demonstrates proper database design principles and efficient data handling techniques.

In this enhancement, I created a MongoDB interface using JavaScript and Express to dynamically manage user-generated recipe reviews. This replaced the static JSON-based system with real-time database interactions. Authenticated users can now create, edit, and delete their own reviews, with all reviews being stored in MongoDB Atlas and served via RESTful API endpoints. This enhancement significantly elevates the project to a full-stack application and showcases practical implementation of database-driven design. The front end is deployed on Netlify, and the backend is hosted on Render.
