Enhancement One

Software Design and Engineering

The Recipe Finder App is a single-page React application originally developed during my time in the General Assembly Software Engineering Immersive. The app allows users to search for meal recipes by name, nationality, or ingredients and provides instructions, ingredients, and optional video links. This particular artifact was selected for my capstone ePortfolio because it provides a clear foundation for demonstrating enhancements in software engineering, particularly around authentication and scalable design patterns.

I selected this artifact because it provided the ideal foundation for enhancing an unauthenticated, public-facing web application with real-world features expected of production-level apps. The enhancement I implemented focused on introducing Google OAuth login, creating a dynamic login-confirmation page, and modifying navigation and review-related features to respond to authentication state. The result was a more secure and interactive user experience that reflects my professional capability to design and deliver industry-standard software solutions.

Through the enhancement process, I demonstrated skills in secure authentication integration, session management, stateful UI logic, conditional rendering, and user interface refinement. I configured token-based authentication using Google OAuth, ensured login state was persisted using browser localStorage, and developed logic to display different views for authenticated and non-authenticated users. These changes are reflected both in the navigation bar—where "Login" or "Logout" are shown depending on the user's session—and in the conditional access to the review submission interface.

During development, I encountered several challenges. One was integrating the Google Identity Services library in a way that worked seamlessly with React's component lifecycle. I also had to address how to securely store user session information and prevent unauthorized access to review features. I handled these challenges through iterative testing, careful debugging, and learning from best practice documentation and community examples. Instructor and peer feedback helped guide refinements—such as adding a login confirmation page for better user experience and ensuring session tokens expire appropriately.

This enhancement allowed me to fully meet the following CS499 course outcomes: (1) Developing a security mindset by integrating and managing OAuth tokens, (2) Using well-founded and innovative tools like Google’s identity services API in combination with React and modern JavaScript, (3) Delivering professional-quality technical solutions with a focus on secure and scalable interface design, and (4) Employing strategies for collaborative environments, which is reflected in authenticated user engagement with the review system. The only course outcome not explicitly demonstrated in this particular enhancement is advanced algorithmic logic, which is the focus of my second artifact.
