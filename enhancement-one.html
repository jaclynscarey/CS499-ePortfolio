Enhancement One

Software Design and Engineering

Software Design and Engineering

Artifact Overview

The artifact I chose for Enhancement One is my Recipe Finder App, a web application initially built during my time at General Assembly’s Software Engineering Immersive bootcamp. The app allows users to search for recipes by nationality, ingredient, or name using data from TheMealDB API. It was developed using React for the front end, and while the original version had basic search functionality and routing, it lacked authentication, user-specific content, and robust state-aware UI components.

Justification for Inclusion in ePortfolio

I selected this artifact because it demonstrates core software design and engineering principles, particularly in front-end architecture, routing, component modularity, and now — after enhancement — authentication and stateful UI logic. It has evolved into a more realistic application, one that mirrors what would be expected of a junior-to-mid-level front-end engineer in a professional environment.

With this enhancement, I added:

  • Google OAuth authentication using @react-oauth/google, enabling users to log in with Google credentials.
  • Conditional navigation logic to display “Login” or “Logout” depending on authentication status.
  • User profile preview in the nav bar (user name and profile picture).
  • A new component /login-confirmation that greets the user after logging in and re-directs to the home page.
  • A reviews section on the recipe details page that only appears for authenticated users.

    • These additions not only improved the user experience but also demonstrate my ability to build secure, user-aware interfaces while managing application state and user identity in a front-end React environment.

    Outcome Alignment and Updates

    This enhancement demonstrates progress toward multiple CS-499 course outcomes:

  • Outcome 1: Collaborative Environments – Implementing Google OAuth simulates real-world identity management, a necessary foundation for collaboration in multi-user applications.
  • Outcome 2: Professional Communication – The app's improved UI/UX and conditional navigation clearly communicate system state to the user.
  • Outcome 4: Innovative Computing Techniques – By integrating modern tools (Google OAuth, token decoding with jwt-decode, and stateful UI components), I showcased practical engineering skills relevant to industry needs.
  • Outcome 5: Security Mindset – OAuth implementation improves security by ensuring that only authenticated users can interact with sensitive or user-specific features like recipe reviews.

    • My outcome coverage plan from Module One remains unchanged. This enhancement addressed exactly what I had planned and laid the foundation for future enhancements (like database connectivity and CRUD operations on reviews).

    Reflection on the Enhancement Process

    Enhancing this app gave me a deeper understanding of secure authentication flows and how to conditionally render UI elements based on auth state — a very common industry task. I encountered challenges integrating the @react-oauth/google package due to its incompatibility with certain React versions. After debugging and reviewing documentation, I resolved this by ensuring that the correct version of React was installed and used jwt-decode to extract user data.
    Additionally, conditionally rendering the nav bar’s content based on login status required state management and persisted storage through localStorage, which helped solidify my understanding of client-side session logic.
    Most importantly, I learned how to refactor my code to make it cleaner, more modular, and more scalable. Instead of cramming all logic into one component, I broke functionality into smaller, maintainable components and hooks. I also cleaned up unused code and reviewed error handling paths.

    Conclusion

    This enhancement meaningfully elevated the quality and realism of my project. It strengthened my front-end engineering capabilities, showcased modern authentication practices, and laid the groundwork for future features that involve user-generated content. As a result, this artifact now reflects not just a school project, but a software product that could realistically be iterated upon in a production environment.
